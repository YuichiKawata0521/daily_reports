# 【要件定義書】AI駆動型 日報・週報自動生成＆管理システム

## 1. プロジェクト概要

日々のPC操作ログや送信メールの内容から、AIを活用して日報および週報のドラフトを自動生成する業務効率化システム。作成されたレポートはユーザーが確認・編集したのち、設定された関係者やチャットツールへ自動送信される。ドメイン駆動設計（DDD）に基づき、複雑な権限管理やステータス遷移を堅牢に実装することを目的とする。

## 2. 技術スタック・アーキテクチャ

* **フロントエンド:** Next.js, TypeScript
* **バックエンド:** Django, Python (Django REST FrameworkによるAPI構築)
* **データ収集クライアント:** Python (exe化された常駐アプリ等によるローカルPCログ収集)
* **設計手法:** ドメイン駆動設計（DDD）
* **データベース設計方針:** 正規化と履歴管理を重視した堅牢なモデリング

## 3. アクターと権限モデル

* **一般メンバー:** 自身の日報・週報の作成、編集、閲覧。同グループ内のレポート閲覧。
* **マネージャー（役職者）:** 自グループおよび、システム管理者から特別に許可された他グループのレポート閲覧。
* **システム管理者:** グループ（部署）の作成、権限の付与、カスタムテンプレートの設定。

## 4. 機能要件

**A. データ収集・統合**
ローカルPCの操作アクティビティを定期的にバックエンドへ送信する仕組みの構築。
OAuth等の連携により、送信済みメール（Gmail等）のテキスト内容を取得。

**B. AI自動生成（LLM連携）**
収集した操作ログとメール内容をコンテキストとしてLLMへ渡し、日報のドラフトを自動生成。
月曜日から木曜日までの日報データを解析・要約し、金曜日に週報のドラフトを自動生成。
日報の内容から既存のシステムタグを自動抽出して付与。該当がない場合は新規タグの作成を提案。

**C. レポート管理とカスタムテンプレート**
生成された日報・週報をユーザーがブラウザ上で確認、手動編集して承認（送信）する機能。
手動での完全なレポート作成およびタグの管理。
グループ（部署）ごとに、レポートのフォーマットやAIへ渡すシステムプロンプトをカスタマイズできるテンプレート機能。

**D. アクセス制御と組織可視化**
所属グループや役職権限を持たないユーザーのレポートは原則非表示とする、厳密なマルチテナント的アクセス制御。
組織図を階層的にドリルダウンし、グループ状況やレポートを直感的に把握できるダッシュボードUI。

**E. 自動通知・外部ツール連携**
レポート承認完了後、事前に設定された上司や同僚のメールアドレスへ自動配信。
レポート作成や特定のイベントをトリガーとして、SlackやTeamsなどの外部チャットツールへWebhook経由で自動通知。

## 5. 開発フェーズ（ロードマップ）

**Phase 1: コア機能・権限モデルの確立（AI・自動化なしの手動運用ベース）**
DDDに基づくデータモデル（ユーザー、グループ、ロール、レポート、タグ、テンプレート）の設計。
Next.jsによる基礎的なUI構築と、Djangoのカスタムユーザーモデルを拡張したアクセス制御の実装。

**Phase 2: AI支援機能の実装（データは手動入力）**
手動入力された箇条書きメモ等からのAI日報生成機能およびタグ提案APIの実装。
保存された日報データ群を元にした週報生成ロジックの構築。
部署ごとのカスタムテンプレートに応じた動的プロンプト処理。

**Phase 3: データ収集の自動化**
PC操作ログ収集プログラムの開発と、バックエンドの受け入れAPIの構築。
メール履歴の連携と、収集データに基づく「完全自動でのレポートドラフト作成」機能の実装。

**Phase 4: 配信・通知連携とUIの高度化**
メール配信機能および外部チャットツール（Slack/Teams）への通知APIの実装。
フロントエンドにおける組織階層のドリルダウン表示など、高度な可視化の導入。
